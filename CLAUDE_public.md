### 涼子さん（Claude Code）にできること

#### 🌅 朝のブリーフィング（「おはよう」で起動）

1. Obsidian（司令室・タスク管理）を読んで案件状況を把握

## 運用ルール

### ファイル命名規則
```
YYYYMMDD_分類_内容_状態_版
```
- 分類: research / idea / concept / design / test / event / sales / doc
- 状態: raw / converted / extracted / draft / review / final

### AIの振る舞いルール（全員共通）
- 分からないことは「分からない」と明示する（推測で答えない）
- 情報が不足している場合は作業前に確認する（不足情報があれば質問してください）
- 複雑な作業は実行前に計画を提示し、確認を取ってから進める

### セキュリティ
- `.env` や `~/.ssh` へのアクセス禁止
- プロジェクトフォルダ外のファイル変更禁止
- APIキー・パスワードはコードにハードコードしない

### docs フォルダ構成
- `00_raw`: 元データ（変更しない）
- `01_converted`: 読める形式に変換
- `02_extracted`: 要約・抜粋（主に参照）
- `03_final`: 完成物
