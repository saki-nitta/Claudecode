# /podcast-batch — ポッドキャストノートを複数本まとめて並列生成する

## 背景

毎週のポッドキャストルーティン（[毎週水曜：オリジナルポッドキャスト作成手順.md](D:\Obsidian_Sync\リモート保管庫1\08_手順\毎週水曜：オリジナルポッドキャスト作成手順.md)参照）で、
テーマ深掘りノートを複数本（基本5本）まとめて作る。
リサーチ・執筆はサブエージェントの中で完結させ、メインの会話には保存報告だけを残す（トークン節約）。

## 対象ファイル

- テーマ候補ストック: `D:\Obsidian_Sync\リモート保管庫1\08_手順\毎週水曜：オリジナルポッドキャスト作成手順.md`
- 保存先: `D:\Obsidian_Sync\リモート保管庫1\05_発信\podcast\`（フォルダが無ければ作成）

## 手順

1. テーマ候補ストックを読む。5カテゴリ（クラファン・販売／木育・おもちゃ市場／子育て支援・地域活動／事業運営・ブランディング／AI・ツール活用）から、未消費（`- [ ]`）の項目を1つずつ選ぶ（計5本）。
   - ユーザーが本数やテーマを指定した場合はそれに従う
   - あるカテゴリの候補が尽きていたら、その旨を最後の報告に含める
2. 今日の日付（YYYYMMDD / YYYY-MM-DD）を確認する。
3. 選んだテーマ数だけ、Agentツールを**1メッセージにまとめて呼び出し、並列実行**する。`subagent_type: general-purpose`。各呼び出しのプロンプトは以下を埋めて自己完結させる（会話の前提は共有されないので、必要な情報は全部書く）：

   ```
   木のおもちゃ事業・地域活動・子育て支援に取り組む事業者「とうめいしっぽ」向けの
   ポッドキャスト深掘りノートを1本作成して保存するタスク。

   テーマ：【{{テーマ}}】

   手順：
   1. Web検索で本テーマに関する直近1ヶ月以内の日本語記事・事例を3件収集する
      （特徴だけでなく「なぜ今注目されているか」も含める）
   2. 以下の構成で4,000字以上のノート本文を書く：
      背景→現状→事例→とうめいしっぽへの示唆→行動提案
   3. Writeツールで以下のパスにファイルを保存する：
      D:\Obsidian_Sync\リモート保管庫1\05_発信\podcast\{{YYYYMMDD}}_doc_podcast-{{テーマ}}_draft.md

      ファイル内容のフォーマット：
      ---
      date: {{YYYY-MM-DD}}
      tags: [ポッドキャスト, {{テーマ}}]
      status: draft
      ---
      # {{テーマ}}

      （本文）

   最後の返答は「保存完了：ファイル名＋1行サマリ」のみにすること。本文をそのまま返さないこと。
   ```

4. 全エージェントの保存が完了したら、テーマ候補ストックの選んだ項目に `- [x]` をつける。
5. ユーザーには保存したファイル名一覧と1行サマリだけを報告する（本文は貼らない）。NotebookLMへのソース追加・音声生成はユーザー側の作業なので涼子は行わない。

## 使い方

ユーザーが「ポッドキャスト5本作って」「今週分のポッドキャストお願い」などと言ったら、確認なしで実行する。
本数や個別テーマの指定があればそれに従う。
