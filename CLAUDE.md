- プロジェクトフォルダ外のファイルの変更

## 命名ルール（厳守）
ファイル名は以下の形式を必ず使用する：

`YYYYMMDD_分類_内容_状態_版`

・分類は定義済みのものから選択する  
（research / idea / concept / design / test / event / sales / doc）

・状態は以下から選択する  
（raw / converted / extracted / draft / review / final）

・内容は短く具体的に統一する

---

## フォルダ運用ルール
/docs 配下は以下で統一：

- 00_raw：元データ（変更しない）
- 01_converted：読める形式に変換
- 02_extracted：要約・抜粋（主に参照）
- 03_final：完成物

※AIは主に「01」「02」を参照する
